Build a habit-tracking + yield-farming dApp called StreakSats on Bitcoin L1 using OP_NET. == CORE CONCEPT == Users stake BTC (minimum 10,000 sats) or PILL tokens (minimum 500 PILL) to make a public commitment to a real-life habit. Every check-in period they call a checkIn() function on-chain to prove they maintained the habit. Maintaining streaks earns yield. Breaking streaks triggers the Impatience Tax. == SMART CONTRACT FUNCTIONS == createHabit(name: string, frequency: string, stakeAmount: u256, token: string) - frequency is either "daily" (86400 second window) or "weekly" (604800 second window) - token is either "BTC" or "PILL" - Transfers stake from user wallet into contract pool - Records: user address, habit name, frequency, stakeAmount, streakCount=0, lastCheckIn=0, isActive=true checkIn(habitId: u256) - Verifiable once per frequency window (enforced by block timestamp) - Increments streakCount by 1 - Updates lastCheckIn to current block timestamp - Emits CheckInEvent(user, habitId, newStreakCount) - If streak hits 7, 30, or 100 days: mint a soulbound OP_721 badge NFT (non-transferable) breakStreak(habitId: u256) - Called automatically-detectable when checkIn window is missed (or callable by anyone to trigger) - Applies Impatience Tax: deducts 10% of user's stakeAmount - Tax is added to the redistributionPool - Remaining 90% of stake is returned to user - Resets streakCount to 0, isActive=false for that habit claimYield() - User claims their proportional share of the redistributionPool - Share formula: userStreakLength / totalActiveStreakLengths * redistributionPool - Only active streak holders can claim - Resets their claimable balance to 0 sponsor() - Any wallet can call this with a BTC transfer to add funds to the redistributionPool - Records sponsor address and amount for display challenge(friendAddress: address, multiplier: u8) - Sends a challenge to a friend's wallet address - multiplier (1–3x the base entry fee) is proposed - Friend has 48 hours (measured in blocks) to accept via acceptChallenge(challengeId) - If accepted: both parties must stake multiplier × their base entry amount - If declined or expired: no penalty getLeaderboard() - Returns top 10 addresses sorted by active streakCount descending - Readable by anyone, no gas cost getMotoMiles(user: address) - Returns the user's motoMiles count (1 mile earned per successful checkIn) - Stored on-chain, readable for airdrop snapshot purposes == SOULBOUND BADGES (OP_721) == Deploy a separate non-transferable OP_721 contract called StreakBadge. Override the transfer() function to always revert (soulbound = wallet-bound forever). Mint badges at these milestones: - 7-day streak → "Consistent" badge (metadata: bronze flame icon, streak count, habit name, date) - 30-day streak → "Dedicated" badge (metadata: silver flame icon) - 100-day streak → "Legend" badge (metadata: gold flame icon) Badge metadata stored on-chain in OP_721 tokenURI as JSON string. == TOKEN SUPPORT == Accept both native BTC (in satoshis) and PILL token (OP_20) as stake currency. If staking PILL: use transferFrom() to pull PILL tokens into the contract on createHabit(). Track separate pools: btcPool and pillPool. Yield from btcPool paid in BTC; yield from pillPool paid in PILL. == MOTO MILES == Track a motoMiles: Map<address, u256> counter. Increment by 1 on every successful checkIn(), regardless of habit currency. Expose getMotoMiles(user) as a public view function. Display on user dashboard — label it "Moto Miles (for airdrop snapshot)". == FRONTEND SCREENS == Build a clean, dark-mode UI with Bitcoin/orange branding. Include these screens: 1. HOME — Hero with tagline "Stack Sats. Build Habits. Earn Yield." Connect Wallet button. Live stats: Total Staked, Active Streamers, Total Redistributed. 2. CREATE HABIT — Form: habit name (text), habit type (dropdown: exercise / reading / orange-pilling friends / no coffee / custom), frequency (daily/weekly toggle), stake amount + currency (BTC or PILL). Preview shows expected yield range. Submit = calls createHabit(). 3. MY DASHBOARD — Active habits list. For each: streak counter (big number), last check-in time, "Check In Now" button (calls checkIn()), yield claimable amount, "Claim Yield" button. Moto Miles counter. Badges earned display. 4. LEADERBOARD — Top 10 table: rank, wallet address (truncated), habit name, streak days, Moto Miles. Refresh button. 5. CHALLENGE — Address input field, multiplier selector (1x/2x/3x), "Send Challenge" button. Pending challenges inbox: accept or decline. 6. BADGES — Grid of all 3 badge types. Earned badges glow orange. Locked badges shown greyed out with requirement. == DEPLOYMENT == Use OP_NET AssemblyScript. Deploy via OP_WALLET Chrome extension. Prioritize: checkIn + Impatience Tax + Leaderboard + Badges first. Challenge and sponsor features second. If any feature is too complex, simplify rather than skip — a working MVP is better than a broken full feature set.

2.2 — Yield Formula

// How much yield a user earns from the redistribution pool: userYield = (userStreakDays / totalActiveStreakDays) × redistributionPool // Example variables: userStreakDays = 30 days totalActiveStreakDays = 480 days (sum of all active streakers) redistributionPool = 50,000 sats (accumulated from Impatience Tax + sponsors) // Result: userYield = (30 / 480) × 50,000 = 3,125 sats
// Impatience Tax on streak break: slashedAmount = stakeAmount × 0.10 // 10% penalty returned = stakeAmount × 0.90 // 90% back to user toPool = slashedAmount // feeds redistributionPool // Example: user staked 10,000 sats, breaks streak on day 5: slashedAmount = 1,000 sats → redistributed to streak-keepers returned = 9,000 sats → back to breaker's wallet

Frontend & UX Flow
Every screen, every user interaction, wireframed

3.1 — Complete User Journey

Step 1 — First Visit
Home Screen — Hook them immediately
  • Bold headline: "Stack Sats. Build Habits. Earn Yield."
  • Live counters: Total BTC Staked, Active Streakers, Total Redistributed today
  • "Connect Wallet" (OP_WALLET) is the single CTA
  • Leaderboard preview below — top 3 streaks visible without connecting
  • Short explainer: How it works in 3 bullet points
Step 2 — Onboarding
Create Habit Screen — Commitment ritual
  • Habit name text field (e.g. "Morning run before 7am")
  • Habit category dropdown: Exercise / Reading / Orange-Pilling Friends / No Coffee / Custom
  • Frequency toggle: Daily (more addictive) / Weekly (cheaper gas)
  • Stake amount slider: 10,000–500,000 sats OR 500–50,000 PILL
  • Preview card updates live: "At this streak level your expected weekly yield is X sats"
  • Submit button: "Make My Commitment" → wallet confirms transaction
Step 3 — Daily Return (the sticky loop)
Dashboard Screen — The core retention surface
  • Big flame emoji + streak number front-and-center ("🔥 Day 14")
  • Countdown timer to next check-in deadline (red when under 2 hours)
  • "CHECK IN NOW" button — large, orange, impossible to miss
  • Yield claimable amount with "Claim" button beside it
  • Moto Miles counter with progress bar to next tier
  • Badges row: earned ones glow, locked ones show X days remaining
  • Quick link to challenge a friend
Step 4 — Social Viral Loop
Challenge Screen — Drive word-of-mouth
  • Paste a Bitcoin wallet address or ENS-style name
  • Select multiplier: 1× / 2× / 3× with impact description
  • "Send Challenge" — sends on-chain challenge tx
  • Challenge inbox: pending received challenges with Accept / Decline
  • 48-hour countdown on pending challenges
  • Share button: generates tweet "I just challenged @[address] to a 3× StreakSats battle #opnetvibecode"
